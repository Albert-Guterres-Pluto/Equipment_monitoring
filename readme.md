# 过滤设备监测建模项目

## 项目简介
2026年北京高校数学建模校际联赛 A 题「过滤设备监测」的完整代码实现。  
涵盖数据预处理、透水率规律分析、寿命预测、维护方案优化及敏感性分析。  
运行 `main.py` 可一键生成所有结果和图表。

## 环境配置（队友必读）
1. **安装 Python 3.9 或以上版本**  
   推荐 [Anaconda](https://www.anaconda.com) 或从 [python.org](https://www.python.org) 下载。
2. **安装依赖库**  
   打开终端，进入项目根目录，执行：
   ```bash
   pip install pandas numpy scipy openpyxl matplotlib
   ```
   如果使用 Anaconda，也可用：
   ```bash
   conda install pandas numpy scipy openpyxl matplotlib
   ```
3. **确认数据文件**  
   确保 `data/raw/` 内包含 `附件1.xlsx` 和 `附件2.xlsx`，否则程序会报错。

## 如何运行
- 在终端的 **项目根目录** 下执行：
  ```bash
  python main.py
  ```
- 等待约 15 秒，所有结果（CSV 和图片）将自动更新在 `results/` 目录下。
- **注意**：不要进入 `src/` 或 `utils/` 目录执行脚本，务必保持在根目录。

## 路径兼容性说明
- 本项目所有路径均基于 **项目根目录自动定位**，无需手动修改任何配置。
- `config.py` 中通过 `Path(__file__).resolve().parent` 确定根目录，其他脚本均使用 `config.PROJECT_ROOT` 构建绝对路径。
- **因此，只要保持整个文件夹结构完整，无论在谁的电脑上、磁盘位置如何，均可直接运行。**

## 常见问题排查
- **报错 `ModuleNotFoundError`**：请检查是否安装依赖，手动安装上述库。
- **报错找不到文件**：确认 `data/raw/` 中有 `附件1.xlsx` 和 `附件2.xlsx`，且终端工作目录为项目根目录。
- **图表中文显示为方块**：字体问题，可在 `plot.py` 中删除 `plt.rcParams['font.sans-serif']` 行，或安装 SimHei 字体。
- **运行 `main.py` 后部分步骤跳过**：查看控制台错误信息，多半是路径或数据缺失。

## 📌 重要更新与讨论（2025-04-25）
### 1. 题目勘误：大维护次数修改
原题中“每年会对每个过滤器进行1至4次不等的大维护”已更正为 **“0至4次”**。  
对应修改：
- `model.py` 中 `get_future_schedule` 函数已调整：若某台设备的历史大维护次数为 0，则未来不再安排大维护；仅有1次历史记录时，默认间隔365天；有2次及以上时取中位数间隔。  
- 此修改已影响部分设备的寿命预测结果（例如 A_8 从 5.57 年降至 5.43 年，A_4 从 7.35 年降至 7.22 年），详见 `results/life_predictions_final.csv`。

### 2. 大维护在优化中为何全部为 0 次？ 
运行第三问优化后，所有设备的最优策略均 **不包含大维护**（大维护次数 = 0）。  
**这不是代码错误，而是当前参数下的合理数学结果**，原因如下：
- **成本效益不对等**：中维护成本 3 万元，平均增益 19.53；大维护成本 12 万元，平均增益仅 16.58。中维护更便宜且增益更高，算法自然回避大维护。
- **损伤惩罚差异**：每次大维护会使日均衰减率额外增加 0.005（中维护为 0.002），进一步削弱大维护的竞争力。
- **物理含义存疑**：从实际经验看，大维护应该带来更大透水率提升，但第一问基于真实数据的统计未能体现这一点。可能原因：大维护常在透水率很低时进行，恢复空间有限；或样本量少导致统计偏差。

**团队讨论要点**：
- 题目给出 0~4 次的上限，暗示大维护在某些情况下是有益的。当前的 0 次结果可能意味着模型低估了大维护的实际价值。
- 若要更贴合题意，可考虑：
  1. **重新标定大维护增益**（例如取维护后 30 天平均透水率与维护前的差值，或人工设定合理增益 ≥ 25）。
  2. **在优化中加入“强制大维护周期”**（如每年至少 1 次），作为对比方案展示成本变化。
  3. **在第四问敏感性分析中增加“大维护增益波动”场景**，观察增益增大时大维护能否被触发，以此说明参数敏感性。
